# 08. チェックアウト画面

**対応JSP**: `src/main/webapp/WEB-INF/jsp/bookstore/sales/checkout.jsp`  
**URL**: `/sales/checkout.do`  
**必要権限**: ログイン済み (カートが空の場合は `/sales/entry.do` へリダイレクト)  
**Actionクラス**: `com.example.bookstore.action.SalesAction`

## 注文サマリーテーブル列

| 列名 | データ | 備考 |
|------|--------|------|
| # | 行番号 | |
| Book | `"Book #" + cartItem.bookId` | ※タイトルが表示されない実装上の問題あり |
| Qty | `cartItem.qty` | |
| **合計** | `session["checkoutTotal"]` | |

---

## 入力項目

| # | 項目名 | name/id | 型 | 必須 | バリデーション |
|---|--------|---------|-----|------|-------------|
| 1 | メールアドレス | `customerEmail` | テキスト | ○ | 形式チェック (英数字・@・.・-のみ) |
| 2 | 配送先氏名 | `shipName` | テキスト | ○ | 空チェック |
| 3 | 住所 | `shipAddress` | テキスト | — | |
| 4 | 都市 | `shipCity` | テキスト | — | |
| 5 | 国 | `shipCountry` | セレクト | — | デフォルト: USA |

---

## ボタン

| 名前 | アクション |
|------|---------|
| ← 販売入力に戻る | `/sales/entry.do?method=entry` |
| 注文確定 | フォームサブミット (POST) → 注文処理 |

---

## バリデーション (クライアント側)

- Email: 必須・形式チェック
- 配送先氏名: 必須
- フォーム二重送信防止 (`formSubmitted` フラグ)
- 確定ボタン: 処理中は `disabled` + ラベル変更

---

## 画面遷移

```
[← 販売入力に戻る]    >>> /sales/entry.do    (07. 販売入力)
[注文確定] 成功       >>> /sales/confirm.do  (09. 注文確認)
[注文確定] エラー     >>> 同画面 + エラーメッセージ
カートが空            >>> /sales/entry.do    (07. 販売入力) ※自動リダイレクト
未ログイン            >>> /login.do          (01. ログイン)
```
